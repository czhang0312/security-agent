# Agent Instructions

This repository may contain hand-written comments added during active development to explain intent, edge cases, or local reasoning.

When editing code in this repo:

- Preserve existing developer comments by default, including informal "notes to self" style comments.
- Do not remove comments just because they seem non-essential, verbose, or stylistically inconsistent.
- If a code change makes an existing comment inaccurate, prefer updating the comment in place instead of deleting it.
- Only remove a comment when it is clearly obsolete, misleading, or directly contradicted by the new code.
- Avoid opportunistic comment cleanup outside the scope of the requested task.

The goal is to protect the developer's working context while still allowing accurate maintenance of comments when code changes require it.
